# Election-Data-Report-Analysis
This project provides a comprehensive analysis of election results, focusing on candidate demographics, party performance, and transparency metrics. Using Excel pivot tables and data cleaning techniques, I transformed raw election data into actionable insights regarding political representation and candidate backgrounds.


1. Candidate Integrity Analysis (Candidate VS criminal cases):

One of the core features of this project is identifying the legal transparency of winning candidates.

The Goal: To see which parties have the highest number of winners with declared criminal cases.

Key Finding: From the analyzed data, we identified 15 winning candidates with a combined total of 81 criminal cases.

Technical Approach: Used Pivot Tables to filter by "Winner" status and then aggregated the count of cases by Party labels.

2. Party Performance Metrics (Political_Party vs Highest_Seat):

This section evaluates which political parties dominated the election cycle.

Metric: Total seats won vs. total candidates fielded.

Insight: Helps identify the "strike rate" of a party—how effectively they convert candidates into winners.

3. Constituency Competition (Constituency vs Candidates):
   
This analysis looks at the density of the election.

Analysis: By mapping the number of candidates against each constituency, we can determine which regions were the most "hotly contested."

Impact: Highlights areas with high political interest versus those with a clear, unopposed lead.

4. Voter Mandate (Highest Votes vs Constituency):
   
Focus: Identifying "Mega-Winners"—candidates who won by the largest margins or received the highest absolute number of votes.

Insight: Distinguishes between a narrow win and a strong public mandate.

Technical Skills Demonstrated

Data Cleaning: Handled inconsistent party naming conventions and managed missing values in the criminal records column.

Pivot Table Architecture: Built dynamic summaries to slice data by Party, Constituency, and Winner status.

Data Transformation: Converted raw LS_2.0 (Lok Sabha) datasets into readable reports for stakeholders.

Aggregation Logic: Used SUM and COUNT functions within Pivot Tables to differentiate between the number of winners and the total number of cases.
